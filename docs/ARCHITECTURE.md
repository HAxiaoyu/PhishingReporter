# 钓鱼邮件上报系统架构文档

## 系统概述

钓鱼邮件上报系统是一个企业级的安全工具，用于收集、分析和存档员工上报的钓鱼邮件。系统由两个主要组件组成：

1. **Outlook VSTO 插件** - 客户端组件，运行在用户的 Outlook Classic 中
2. **ASP.NET Core Web API** - 服务端组件，处理上报数据、存档和分析

## 数据流

### 上报流程

```
1. 用户选择邮件 → 点击"上报钓鱼"按钮
                │
                ▼
2. 插件显示确认对话框
   - 显示邮件摘要
   - 用户可添加备注
   - 确认/取消
                │
                ▼
3. 插件提取邮件信息
   - EmailExtractor.ExtractReport()
   - 提取发件人、收件人、主题
   - 提取邮件头 (SPF, DKIM, DMARC)
   - 提取附件信息
   - 导出原始 EML
                │
                ▼
4. 插件发送到后端 API
   - POST /api/v1/reports
   - JSON payload
   - HTTPS 加密传输
                │
                ▼
5. 后端处理
   a) 保存到数据库
   b) 存储原始 EML 文件
   c) 存档到 Exchange (异步)
   d) 运行钓鱼分析
   e) 发送通知 (异步)
                │
                ▼
6. 返回结果给用户
   - 成功/失败消息
   - 可选：标记邮件
```

## 组件说明

### Outlook 插件 (PhishingReporter)

| 组件 | 文件 | 功能 |
|------|------|------|
| ThisAddIn | ThisAddIn.cs | VSTO 入口点，初始化服务，注册右键菜单 |
| ReportRibbon | Ribbon/ReportRibbon.cs | 功能区按钮 UI 和事件处理 |
| ReportDialog | Forms/ReportDialog.cs | 上报确认对话框 |
| EmailExtractor | Services/EmailExtractor.cs | 邮件信息提取，EML 导出 |
| ReportService | Services/ReportService.cs | HTTP 上报到后端 |
| ConfigManager | Services/ConfigManager.cs | 读取和管理配置 |

### 后端 API (PhishingReporter.Backend)

| 组件 | 项目 | 功能 |
|------|------|------|
| ReportsController | Api | 处理上报 API 请求 |
| ReportService | Core | 处理上报业务逻辑 |
| AnalysisService | Core | 钓鱼特征分析 |
| EmailArchiveService | Infrastructure | Exchange EWS 存档 |
| NotificationService | Infrastructure | 邮件/Teams 通知 |
| FileStorageService | Infrastructure | EML 文件存储 |
| ReportRepository | Infrastructure | 数据库操作 |

## 安全设计

### 认证

- **插件 → API**: API Key 认证 (Header: X-API-Key)
- **API → Exchange**: Windows 集成认证或服务账号

### 传输安全

- 所有通信使用 HTTPS (TLS 1.2+)
- 后端 API 配置 SSL 证书

### 数据安全

- 敏感配置加密存储
- 数据库连接字符串加密
- 日志脱敏（不记录敏感信息）

### 访问控制

- 用户只能查看自己上报的邮件
- 安全团队有查看所有上报的权限

## 扩展性考虑

### 水平扩展

- API 无状态设计，支持多实例部署
- 使用负载均衡器分发请求

### 功能扩展

- 分析规则可配置化
- 支持添加更多分析器
- 支持 SIEM 集成

### 数据保留

- 配置化的数据保留策略
- 自动清理过期数据
